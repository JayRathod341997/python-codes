# Project 10 — Sales Analysis: Loops vs Comprehensions

## The Real-Life Problem

A retail analytics team has 20 transactions (product, category, qty, price, region, salesperson). They need to:
- Filter transactions by category
- Compute revenue per region
- Invert dicts (region → salespeople)
- Use generators for lazy evaluation

**Project 10 is a "REFACTOR" edition:**
- Section 1: Traditional for-loops (as Projects 04/05 did)
- Section 2: Same operations using comprehensions
- Shows the power of Pythonic code

## Domain Context

**Industry**: Retail / E-commerce / Business Intelligence
**Tools**: pandas, Spark, BI dashboards
**Why Python**: Comprehensions make data manipulation concise and readable

## Concepts

List comprehensions, dict comprehensions, set comprehensions, generator expressions, nested comprehensions, dict operations
